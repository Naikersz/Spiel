# Как запустить проект

## Правильный файл для запуска

**Запускайте проект через файл `game/main.py`:**

```bash
python game/main.py
```

**Важно:** Запускайте из корневой директории проекта (где находится папка `game`), не изнутри папки `game`.

**НЕ используйте:**
- Старый файл `main.py` в корне проекта (удален)
- Устаревшие модули `engine.py`, `create_character.py`, `hero_classes.py` (удалены)

## Что было исправлено

### Проблема: Кнопка "Start Game" ничего не делала

**Причина:** При нажатии "Start Game" состояние менялось на `PLAYING`, но не было логики для отрисовки игрового процесса.

**Решение:** Добавлена обработка состояния `PLAYING` в `GameInitializer`:
- При переходе в `PLAYING` создается игровой процесс
- Обновляются игровые объекты (игрок, враги)
- Отрисовывается игровой мир вместо меню

## Как это работает

1. **Запуск:** `main.py` → `GameLauncher` → `GameInitializer`
2. **Меню:** `MenuManager` управляет состояниями (MENU, PLAYING, PAUSED)
3. **Игра:** При `PLAYING` создается игровой процесс с персонажем и врагами
4. **Управление:**
   - WASD - движение
   - ESC - пауза
   - 1-7 - переключение одежды (если реализовано)

## Структура запуска

```
game/main.py
  └── GameLauncher
       └── GameInitializer
            ├── MenuManager (меню)
            └── GameInstance (игровой процесс при TOWN/PLAYING)
```

## Отладка

Если игра не запускается:
1. Проверьте, что вы в корневой директории проекта
2. Убедитесь, что установлен pygame: `pip install pygame`
3. Проверьте консоль на наличие ошибок

Если "Start Game" не работает:
- Проверьте, что файл `game/core/game_initializer.py` содержит метод `_start_game()`
- Убедитесь, что состояние меняется на `PLAYING` (можно добавить print в `handle_action`)
